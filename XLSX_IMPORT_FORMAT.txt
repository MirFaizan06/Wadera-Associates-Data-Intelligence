WADERA ASSOCIATES — DATA IMPORT XLSX FORMAT
===========================================

REQUIRED COLUMN LAYOUT
-----------------------

Col A  |  Col B   |  Col C (optional)
-------+----------+--------------------
Date   |  Value   |  Note
-------+----------+--------------------

RULES
-----
1. Row 1 MUST be the header row:    Date | Value | Note
2. Data starts at Row 2.
3. Col A (Date):
   - Format: YYYY-MM  (e.g. 2023-01, 2024-06)
   - Excel date cells also accepted (the importer reads them as ISO and takes YYYY-MM)
   - One row per month. Duplicate months for the same dataset are upserted (last write wins).
4. Col B (Value):
   - A plain numeric value in the dataset's default unit (e.g. VND/KWH, USD/tonne, MW, etc.)
   - Do NOT include currency symbols, commas, or formulas.
5. Col C (Note):
   - Optional free-text annotation for that data point.
   - Leave blank if not needed.
6. No extra columns, merged cells, or empty leading columns.
7. Sheet name does not matter — the importer always reads the first sheet.


EXAMPLE — Vietnam Industrial Electricity Price (VND/KWH)
---------------------------------------------------------
| Date     | Value | Note            |
|----------|-------|-----------------|
| 2023-01  | 1650  |                 |
| 2023-02  | 1650  |                 |
| 2023-03  | 1755  | Tariff revision |
| 2023-04  | 1755  |                 |
| 2023-05  | 1920  | New tariff      |
| ...      | ...   | ...             |


HOW THE SAMPLE FILE PROVIDED DIFFERS
--------------------------------------
File: "SAMPLE DATA Vietnam Industrial Electricity Prices by Month.xlsx"

Issue: Columns A and B are EMPTY. Actual data starts at Col C:
  Col C = Month (date)   ← should be Col A
  Col D = VND/KWH        ← should be Col B
  Col E = USD/KWH formula (not needed)
  Col F = USD to VND rate (not needed)

Fix: In Excel, move Month to Col A, move VND/KWH to Col B, delete Cols C–F.
The file "SAMPLE_CORRECT_FORMAT.xlsx" in this folder shows the correct layout.


ACCEPTED DATE FORMATS (auto-detected)
--------------------------------------
  2023-01-01   →  treated as Jan 2023
  2023-01      →  Jan 2023
  01/01/2023   →  Jan 2023 (Excel serial dates also work)


UNIT NOTE
---------
The "Value" column must always be in the dataset's DEFAULT UNIT as set during
dataset creation in the admin panel. Unit conversions (e.g. VND → USD) are
handled automatically by the platform using live exchange rates.
